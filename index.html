关于react-redux

Veröffentlicht am 2016-10-08

一：使用redux的原因：随着js单页面应用开发日趋复杂，js需要管理许多state。如果一个model变化会引起另一个model变化，那么当view变化时，就可能引起对应的model及另一个model的变化，依次的，可能再引起另一个model的变化，这样，state就会很混乱，而redux则是试图让state的变化变得可预测。

二：三大原则

1. 单一数据源

整个应用的state被存在一颗object tree中，并且这个object tree只存在于唯一一个store中。

2. state是只读的

唯一改变state的方法是触发action，action是一个用于已发生事件的普通对象。

这样确保了视图和网络请求都不能直接修改state他们只能表达想要修改的意图。因为所有的修改都被集中化处理，且严格按照一个接一个的顺序执行。

3. 使用纯函数来执行修改

为了描述action如何改变state tree，需要编写Reducer

Reducer只是一些纯函数，接收原来的state以及action，返回新的state

react组件生命周期

Veröffentlicht am 2016-10-07

一：实例化

1. getDefaultProps

每个组件的getDefaultProps只会调用一次，它返回的是一个对象。

2. getInitialState

每个组件的实例会调用一次这个方法，并且只调用一次。这个方法可以初始化每个实例的state。它返回的也是一个对象。

3. componentWillMount

这个方法会在第一次渲染之前被调用，可以在这个方法里面修改state，并且这是render方法调用之前可以修改state的最后机会。

4. render

它是组件类中唯一一个必须的方法，它创建一个虚拟的DOM，用来表示组件的输出。render方法满足下面几点：

  1. 只能通过this.props和this.state访问数据
  2. 可以返回null,false,或者任何React组件
  3. 只能出现一个顶级组件

5. componentDidMount

在这个方法内部可以通过this.getDOMNode()方法访问到真实DOM，因为render方法已经成功渲染出真实的DOM了。

二：存在期

6. componentWillReceiveProps

父组件改变子组件的属性时，componentWillReceiveProps会被调用。可以在这个方法里更改state，以出发render方法重新渲染组件。

7. shouldComponentUpdate

如果确定组件的props或者state的改变不需要重新渲染，可以通过在这个方法里返回false来阻止组件的重新渲染，这样可以提高速度。如果返回false，那么render(), componentWillUpdate(), componentDidUpdate() 都不会被执行。

8. componentWillUpdate

和componentWillMount类似，组件会在接收到新的props或者state进行渲染之前，调用这个方法。不要在这个方法中更新state或者props，而应该在componentWillReceiveProps中更新。

9. componentDidUpdate

和componentDidMount类似,该方法给了我们更新已经渲染好的DOM的机会。

三： 销毁&清理期

10. componentWillUnmount

每当React使用完一个组件时，这个组件必须从DOM中卸载后被销毁，此时componentWillUnmount会被执行，完成所有的清理和销毁工作，在componentDidmount中添加的任务都需要在该方法中撤销，如创建的计时器或者事件监听。

关于bind

Veröffentlicht am 2016-10-07

一：bind与call和apply相似之处

第一个参数是作为执行时函数上下文的this对象，从第二个参数开始至最后一个参数代表的都是函数执行时所需的参数

二：bind独特之处

1. bind返回值是函数

function func(a,b){
 console.log(a+b);
 }
 var obj = 'hhh';
 var a = func.bind(obj,3,4);
 a();
 func.call(obj,4,5);

bind返回之后的函数需要执行一下才能出结果，即bind返回的是函数，而call和apply返回的是结果

2. 后面的参数使用也有区别


 function f(a,b,c){
    console.log(a,b,c);
}
var f_Extend = f.bind(null,”extend_A”);
f(“A”,”B”,”C”)  //这里会输出–> A B C
f_Extend(“A”,”B”,”C”)  //这里会输出–> extend_A A B
f_Extend(“B”,”C”)  //这里会输出–> extend_A B C
f.call(null,”extend_A”) //这里会输出–> extend_A undefined undefined

call是把第二个及以后的参数作为f的实参传进去的，而bind虽说也是获取第二个及以后的参数用于方法的执行，但是f_Extend中传入的实参是在bind中传入参数的基础上往后排的。

es6语法和技巧

Veröffentlicht am 2016-09-27

一：let和const命令

1.let命令

  1. let不像var那样发生“变量提升”，所以一定要在申明后使用，否则会报错。
    console.log(foo); // 输出undefined
console.log(bar); // 报错ReferenceError
var foo = 2;
let bar = 2;
  2. 暂时性死区
    如果区块中存在let和const命令，这个区块对这些命令申明的变量，从一开始就形成封闭的作用域，凡是在这些申明之前使用这些变量的，都会报错。
    暂时性死区的本质是，只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到申明变量的那一行代码出现的时候，才可以获取和使用该变量。
  3. let不允许在相同的作用域内重复申明同一个变量，因此不能在函数内部重新申明函数参数。
    function a(arg){
 let arg = 8;
 console.log(arg);
}
a();//Identifier 'arg' has already been declared

2.块级作用域

  1. es5中没有块级作用域，es6中有块级作用域。
  2. es5中函数只能在顶级作用域和函数作用域中申明，不能在块级作用域中申明。es6允许在块级作用域中申明函数，并且es6规定，块级作用域中，函数申明语句行为类似于let，在块级作用域之外不可引用。
  3. 考虑到环境导致的行为差异太大，应该避免在块级作用域中申明函数。如果确实需要，也应该写成函数表达式，而不是函数申明。

3.const命令

  1. const申明一个只读的常量，一旦申明，就不能改变其值。const变量一旦申明，就必须初始化，不能申明后再初始化。
  2. const的作用域域let相同：只在申明所在的块级作用域内有效。const命令申明的常量也是不提升的，同样存在暂时性死区，只能在申明的位置后面使用。

4.全局对象的属性和全局变量

二：解构赋值

1.如果解构不成功，变量的值就是undefined
2.数值和布尔值结构时，会先转换为对象.
3.解构赋值的规则是，只要等号右边的值不是对象，就先将其转换为对象。由于undefined 和 null无法转换为对象，所以对它们进行解构赋值，就会报错。
4.注意函数参数解构使用默认值和函数参数使用默认值的区别。函数参数结构使用默认值是只要传进来的不是完整的对象就使用默认解构获得默认参数，而函数参数使用默认值则是：如果传入了参数，则使用被传入的参数，不管这个参数对象是否完整；如果没有传入参数才会使用默认参数，并对默认参数的这个对象进行解构。


     function move({x = 0, y = 0} = {}) {
          return [x, y];
        }
        move({x: 3, y: 8}); // [3, 8]
        move({x: 3}); // [3, 0]
        move({}); // [0, 0]
        move(); // [0, 0]
        function move({x, y} = { x: 0, y: 0 }) {
          return [x, y];
        }
        move({x: 3, y: 8}); // [3, 8]
        move({x: 3}); // [3, undefined]
        move({}); // [undefined, undefined]
        move(); // [0, 0]

三：不能使用圆括号的情况

  1. 变量申明语句不能带有圆括号(函数参数也属于变量申明，所以函数参数中不能有圆括号)
  2. 赋值语句中，不能将整个模式或嵌套模式中的一层，放在圆括号中
  3. 可以使用圆括号的情况只有一种：赋值语句的非模式部分
