es6语法和技巧

Veröffentlicht am 2016-09-27

一：let和const命令

1.let命令

  1. let不像var那样发生“变量提升”，所以一定要在申明后使用，否则会报错。
    console.log(foo); // 输出undefined
console.log(bar); // 报错ReferenceError
var foo = 2;
let bar = 2;
  2. 暂时性死区
    如果区块中存在let和const命令，这个区块对这些命令申明的变量，从一开始就形成封闭的作用域，凡是在这些申明之前使用这些变量的，都会报错。
    暂时性死区的本质是，只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到申明变量的那一行代码出现的时候，才可以获取和使用该变量。
  3. let不允许在相同的作用域内重复申明同一个变量，因此不能在函数内部重新申明函数参数。
    function a(arg){
 let arg = 8;
 console.log(arg);
}
a();//Identifier 'arg' has already been declared

2.块级作用域

  1. es5中没有块级作用域，es6中有块级作用域。
  2. es5中函数只能在顶级作用域和函数作用域中申明，不能在块级作用域中申明。es6允许在块级作用域中申明函数，并且es6规定，块级作用域中，函数申明语句行为类似于let，在块级作用域之外不可引用。
  3. 考虑到环境导致的行为差异太大，应该避免在块级作用域中申明函数。如果确实需要，也应该写成函数表达式，而不是函数申明。

3.const命令

  1. const申明一个只读的常量，一旦申明，就不能改变其值。const变量一旦申明，就必须初始化，不能申明后再初始化。
  2. const的作用域域let相同：只在申明所在的块级作用域内有效。const命令申明的常量也是不提升的，同样存在暂时性死区，只能在申明的位置后面使用。

二：解构赋值

1.如果解构不成功，变量的值就是undefined
2.数值和布尔值结构时，会先转换为对象.
3.解构赋值的规则是，只要等号右边的值不是对象，就先将其转换为对象。由于undefined 和 null无法转换为对象，所以对它们进行解构赋值，就会报错。
4.注意函数参数解构使用默认值和函数参数使用默认值的区别。函数参数结构使用默认值是只要传进来的不是完整的对象就使用默认解构获得默认参数，而函数参数使用默认值则是：如果传入了参数，则使用被传入的参数，不管这个参数对象是否完整；如果没有传入参数才会使用默认参数，并对默认参数的这个对象进行解构。


     function move({x = 0, y = 0} = {}) {
          return [x, y];
        }
        move({x: 3, y: 8}); // [3, 8]
        move({x: 3}); // [3, 0]
        move({}); // [0, 0]
        move(); // [0, 0]
        function move({x, y} = { x: 0, y: 0 }) {
          return [x, y];
        }
        move({x: 3, y: 8}); // [3, 8]
        move({x: 3}); // [3, undefined]
        move({}); // [undefined, undefined]
        move(); // [0, 0]

三：不能使用圆括号的情况

  1. 变量申明语句不能带有圆括号(函数参数也属于变量申明，所以函数参数中不能有圆括号)
  2. 赋值语句中，不能将整个模式或嵌套模式中的一层，放在圆括号中
  3. 可以使用圆括号的情况只有一种：赋值语句的非模式部分
