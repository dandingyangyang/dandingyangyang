一：bind与call和apply相似之处

第一个参数是作为执行时函数上下文的this对象，从第二个参数开始至最后一个参数代表的都是函数执行时所需的参数

二：bind独特之处

1. bind返回值是函数

function func(a,b){
 console.log(a+b);
 }
 var obj = 'hhh';
 var a = func.bind(obj,3,4);
 a();
 func.call(obj,4,5);

bind返回之后的函数需要执行一下才能出结果，即bind返回的是函数，而call和apply返回的是结果

2. 后面的参数使用也有区别


 function f(a,b,c){
    console.log(a,b,c);
}
var f_Extend = f.bind(null,”extend_A”);
f(“A”,”B”,”C”)  //这里会输出–> A B C
f_Extend(“A”,”B”,”C”)  //这里会输出–> extend_A A B
f_Extend(“B”,”C”)  //这里会输出–> extend_A B C
f.call(null,”extend_A”) //这里会输出–> extend_A undefined undefined

call是把第二个及以后的参数作为f的实参传进去的，而bind虽说也是获取第二个及以后的参数用于方法的执行，但是f_Extend中传入的实参是在bind中传入参数的基础上往后排的。
